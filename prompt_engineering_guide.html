Prompt Engineering: A Practical Guide

Prompt engineering is the skill of giving AI clear instructions so it produces the output you want. Think of it as writing a good specification.

Basic Formula

Task + Context + Constraints + Output Format

Weak Prompt

Write about climate change.

Better Prompt

Explain climate change to a 12-year-old in under 200 words. Use simple language and include one real-world example.

Key Lessons

  1. Assign a Role: "You are an experienced Python instructor."
  2. Provide Context: Explain why you're asking.
  3. Specify the Output: Bullet points, table, JSON, Markdown, etc.
  4. Add Constraints: Word limit, audience, tone, style.
  5. Use Examples (Few-Shot): Show desired input/output patterns.
  6. Break Complex Tasks: Split large tasks into smaller steps.
  7. Request Structured Thinking: Ask for requirements, options, comparison, recommendation.
  8. Iterate: Improve prompts with more context and clearer instructions.

Advanced Techniques

  • Use delimiters to separate context.
  • Ask for multiple alternatives.
  • Review from different personas.
  • Request self-review before the final answer.
  • Use evaluation prompts with scoring criteria.

Common Mistakes

  • Being too vague.
  • Not providing context.
  • Not specifying an output format.

Checklist

  • Clear goal
  • Sufficient context
  • Defined output format
  • Relevant constraints
  • Examples if helpful
  • Break down complex tasks

Practice Exercises

  1. Rewrite the prompt: "Help me study."
  2. Create a prompt for a 7-day beginner home workout plan.
  3. Write a prompt for a professional project proposal with timeline and budget.

Key Takeaway

High-quality prompts clearly communicate what you want, why you want it, how it should be delivered, and any important constraints.
