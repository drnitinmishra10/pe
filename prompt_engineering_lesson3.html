Prompt Engineering Course — Lesson 3: Advanced Prompt Engineering

This lesson covers advanced concepts including system prompts, context management, Retrieval-Augmented Generation (RAG), prompt optimization, debugging, hallucination reduction, and reusable prompt templates.

1. System Prompts vs. User Prompts

System prompts define the AI's long-term behavior.

You are an expert programming tutor.
Always explain concepts step by step.
Never provide unsafe code.

User prompts define the specific task.

Explain Python decorators with examples.

2. Context Windows

The context window is the maximum amount of information the AI can process in one conversation, including previous messages, uploaded content, and responses.

  • Keep prompts focused.
  • Summarize long conversations.
  • Repeat important details when needed.

3. Tokens

AI processes text as tokens rather than words.

Prompt engineering is fun.
  • Larger prompts use more tokens.
  • Larger responses also consume tokens.
  • Efficient prompts allow more room for useful output.

4. Retrieval-Augmented Generation (RAG)

  1. User asks a question.
  2. Relevant documents are retrieved.
  3. The retrieved content is provided to the AI.
  4. The AI generates an answer based on that content.

Benefits: More accurate, up-to-date, and organization-specific answers.

5. Prompt Optimization

Improve prompts by adding context, constraints, and formatting while removing unnecessary words.

Before:
Write about AI.

After:
Explain how large language models work for university students in under 500 words.
Include one diagram description and three practical examples.

6. Prompt Debugging

  • Simplify the prompt.
  • Test one requirement at a time.
  • Add constraints gradually.
  • Compare multiple versions.

7. Hallucination Reduction

  • Provide source material.
  • Ask the AI to distinguish facts from assumptions.
  • Request citations when appropriate.
  • Allow the AI to say "I don't know."
  • Verify important information independently.

9. Prompt Libraries

Create reusable prompts for recurring tasks such as email writing, code reviews, resume feedback, lesson planning, marketing, and data analysis.

10. Prompt Versioning

v1: Summarize the report.

v2: Summarize the report in 200 words using bullet points.

v3: Summarize the report in 200 words using bullet points,
highlight risks, and end with recommendations.

Best Practices

  • Start with a clear objective.
  • Provide sufficient context.
  • Specify the output format.
  • Add realistic constraints.
  • Test and refine prompts.
  • Reuse successful templates.
  • Verify important information.

Practice Exercises

  1. Write a system prompt for an AI coding tutor.
  2. Create a reusable research-summary prompt template.
  3. Design a RAG workflow for an employee handbook.
  4. Improve a vague prompt with context and constraints.
  5. Create and compare three versions of the same prompt.
Key Takeaways
  • System prompts define long-term behavior.
  • User prompts define specific tasks.
  • Context windows and tokens affect prompt capacity.
  • RAG improves accuracy with external knowledge.
  • Optimization and debugging improve prompt quality.
  • Templates, libraries, and versioning improve efficiency.
